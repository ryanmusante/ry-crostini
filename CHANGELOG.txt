CHANGELOG
=========

crostini-setup-duet5.sh — Crostini post-install bootstrap
Lenovo IdeaPad Duet 5 Chromebook (82QS0001US / 13Q7C6)


2.5.1  2026-03-14
------------------

  Post-audit fixes:

  - Fix M-1: wrap VS Code download/install block in DRY_RUN guard.
    mktemp created a real temp file in /tmp during --dry-run, violating
    the zero-side-effects contract. Also add -s (non-empty) check
    alongside -f before dpkg to prevent installing a 0-byte file on
    download failure.
  - Fix M-2: replace fragile find -not -name snd with -mindepth 1 in
    both step 7 audio verification and step 19 summary. The old filter
    excluded /dev/snd itself by matching basename, but would also
    silently exclude any child named snd.
  - Fix L-1: add -u (nounset) to run_shell() bash -c invocation so
    the subshell enforces set -euo pipefail, consistent with the
    parent script's strict mode.
  - README: update script line count to 1509.

2.5.0  2026-03-13

  Exhaustive audit — complete --dry-run isolation + error handling:

  - Fix H-1: guard all step 2 interactive prompts (open_chromeos_url,
    sleep, read) with DRY_RUN checks across 6 sub-sections: GPU flag,
    microphone, USB passthrough, shared folders, port forwarding, and

  - Fix M-2: add error handling to all unguarded sudo calls that would
    abort the script under set -e without a useful message: sudo ln -sf
    for fd/bat symlinks (|| true), sudo mkdir -p /etc/apt/keyrings

  - Fix M-3: step 8 banner now reads "GTK 2/3/4" to match README, help

  - Fix L-1: --help step 8 description now lists GTK 2/3/4 (was GTK3/4),
    matching actual config files written, banner, and README.
  - Fix L-2: guard python3 --version, pip --version, and git --version
    subshells with || echo fallback to prevent set -e abort when tools
    are missing after partial install.
  - README: update --dry-run feature description to include interactive
    prompts and ChromeOS URL opens; update script line count.

2.3.1  2026-03-08

  Post-audit fixes:

2.3.0  2026-03-08
------------------

  Crostini-exclusive audit — remove everything non-Crostini:

  - Remove pulseaudio daemon package (Crostini bridges audio from host;
    daemon would start conflicting server). Keep pulseaudio-utils only.
  - Remove xwayland package (sommelier provides its own XWayland bridge;
    standalone package conflicts)
  - Fix OLED font rendering: change rgba=rgb to rgba=none across GTK 2/3/4,
    Xresources, and fontconfig. OLED has no LCD subpixel stripe — grayscale
    AA is correct. Remove all lcdfilter=lcddefault settings.
  - Replace ping network check with curl (ICMP is blocked in Crostini VM)
  - Add --yes to gpg --dearmor for NodeSource key (idempotent on re-run)
  - Remove transitional apt-transport-https (built into apt on Bookworm)
  - Bare-metal audit: confirmed zero systemctl, NetworkManager, modprobe,
    bootloader, bluetooth, or display server management references
