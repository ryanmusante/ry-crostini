2.5.5  2026-03-15
------------------

  Post-audit fixes (AUDIT-2.5.5.txt):

  - Fix H-1: add || true guard to `rmdir "$LOCK_FILE"` in cleanup()
    trap handler. The rmdir is a bare command (not in && or || list),
    so when it fails under set -e the trap aborts early — skipping
    the remaining cleanup logic and corrupting the exit code.
    Reproduced: --help, --version, and --reset all exit with code 1
    instead of 0 because the lock was never acquired (created after
    argument parsing) so rmdir fails and set -e fires in the EXIT
    trap before `exit "$rc"` executes.
  - Fix M-1: add || true guard to VS Code curl download (step 14).
    Without it, a download failure aborts the script under set -e
    before reaching the [[ -f ]] && [[ -s ]] check that has the
    user-friendly "Install manually" fallback message.
  - Fix M-2: restructure VS Code dpkg/apt-fix chain from bare
    `run dpkg || run apt -f` + unconditional success log into an
    if/else block. Previously, if both dpkg and apt -f failed, the
    second run() (command after final ||) triggered set -e, aborting
    the script for a non-critical package. Now: success logs on
    install, warns with manual install URL on failure.
  - README: update script line count to 1525.
