CHANGELOG
=========

ry-crostini.sh -- Crostini post-install bootstrap
Lenovo IdeaPad Duet 5 Chromebook (82QS0001US / 13Q7C6)

7.6.3  2026-03-30

  Changes:
  - remove systemd v257.* APT version pin
    (/etc/apt/preferences.d/pin-systemd): systemd upgrades are no
    longer blocked; pin was originally added to prevent v258 cgroup v1
    incompatibility but is no longer desired
  - add version number to run-x86 --version and gog-extract --version
    output; version is baked in at generation time via @@VERSION@@
    placeholder sed replacement (single-quoted heredoc prevents direct
    shell expansion)

7.6.2  2026-03-30

  Fixes:
  - fix 4 local-outside-function crashes introduced in 7.6.1: bash
    rejects local outside a function (exit 1), which under set -e kills
    the script immediately; affected Bookworm→Trixie sources.list.d
    iteration (L989), retroarch.cfg swapchain upgrade (L1876),
    retroarch.cfg frame-delay upgrade (L1900), and scummvm.ini chorus
    upgrade (L1939); replaced with direct assignment + unset
  - fix find|wc-l newline-unsafe count in 3 sites (shared folder count
    L859, audio device count L1349 and L2293): replace with
    find -printf '.'|wc -c which counts entries not lines
  - harden gog-extract .sh handler: validate makeself archive signature
    (grep -qi 'makeself' in first 20 lines) before bash invocation;
    previously any .sh file would be executed with --noexec passed as $1
    (a bash argument, not a makeself flag) to a non-makeself script

7.6.1  2026-03-30

  Fixes:
  - fix header sudo count: ~65 → ~70 (actual runtime count on standard
    Bookworm→Trixie first run; CHANGELOG 6.0.1 already audited ~70 but
    header was never updated)
  - fix _gpu_conf_content() comment: "Called by step 6" → "Called by
    step 5"; function is called exclusively inside the step 5 block
  - fix non-atomic retroarch.cfg upgrade paths: sed -i (swapchain 2→3)
    and printf >> (video_frame_delay_auto append) replaced with atomic
    tmpfile+mv and read+write_file_private respectively; crash mid-write
    could previously corrupt the file
  - fix non-atomic scummvm.ini upgrade path: printf >> (chorus append)
    replaced with read+write_file_private atomic pattern
  - fix gog-extract: chmod +x on user's original .sh installer replaced
    with bash invocation (bash "$installer" --noexec); no longer mutates
    the input file's permissions
  - fix unquoted associative array subscripts in check_tool(): quote
    _TOOL_VER_FLAG[$cmd] in [[ -v ]] and ${} expansion (3 sites);
    safe in practice (tool names contain no glob chars) but inconsistent
    with quoting discipline elsewhere
  - fix potential failglob crash in step 2 sources.list.d glob iteration:
    add nullglob guard around for loop over *.list *.sources; restore
    previous nullglob state after loop

7.6.0  2026-03-29

  Gaming optimizations (spec: ry-crostini-gaming-optimizations v1.2.0):

  Mesa:
  - add MESA_DISK_CACHE_DATABASE_NUM_PARTS=10 to gpu.conf; reduces
    database partition overhead from default 50 on 4 GB RAM
  - add upgrade path: rewrite gpu.conf if NUM_PARTS absent

  PipeWire:
  - add mem.allow-mlock=true to PipeWire core gaming config; allows
    real-time memory locking for audio threads
  - add upgrade path: rewrite PipeWire gaming config if
    mem.allow-mlock absent
  - add WirePlumber ALSA tuning config
    (~/.config/wireplumber/wireplumber.conf.d/51-crostini-alsa.conf):
    period-size=256, period-num=3, headroom=256, disable-batch=true,
    suspend-timeout=0; WirePlumber 0.5+ JSON .conf format (Trixie)

  RetroArch:
  - change video_max_swapchain_images from 2 to 3; reduces virgl
    serialization stalls
  - add video_frame_delay_auto=true; dynamic safe frame delay
    (available since RA 1.9.13)
  - update audio_driver comment: document alsa as fallback for
    RA#17685 (PipeWire quantum hardcoding in Trixie RA 1.20.0;
    fixed in RA 1.21+); default remains pipewire
  - add upgrade paths for existing retroarch.cfg: sed swapchain
    2→3, append video_frame_delay_auto if absent

  ScummVM:
  - add fluidsynth_chorus_activate=false to scummvm.ini; saves
    5–8% CPU on SC7180P by disabling chorus DSP effect
  - add upgrade path for existing scummvm.ini: append
    fluidsynth_chorus_activate=false if absent

  Container:
  - add journald volatile storage config
    (/etc/systemd/journald.conf.d/volatile.conf): Storage=volatile;
    writes journal to RAM only, saves eMMC I/O; step 9d

  Verification:
  - add check_config for WirePlumber ALSA tuning (step 6)
  - add check_config for journald volatile (step 9)
  - add check_config for ry-crostini-cros-pin.service (step 2;
    was in README generated files table but never verified)

  Documentation:
  - update step 6 description: add WirePlumber ALSA tuning
  - update step 9 description: add journald volatile
  - update README generated files table: add WirePlumber ALSA and
    journald volatile entries
  - renumber step 9 subsections: 9d=journald, 9e=env profile,
    9f=XDG dirs
