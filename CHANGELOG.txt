3.8.3  2026-03-16

  * fix _strip_log_ansi sed pattern: \?* is ambiguous in GNU BRE (\? is
    a quantifier, not a literal question mark), so DEC private mode
    sequences like \e[?25h (show/hide cursor) were not stripped from the
    log file.  Fix: replace \?* with character class [?]* which is
    unambiguous across all sed implementations.
  * fix df stderr leak in step 1d (line 548): df could emit warnings to
    stderr (e.g. on unusual filesystems) that leaked to the terminal.
    Steps 2f and 18 already suppressed this with 2>/dev/null — now
    consistent across all three df call sites.
  * fix signal exit codes: cleanup() exited with $? from the failing
    command, not 128+signum.  Parent processes (CI runners, wrapper
    scripts) could not distinguish a signal death (SIGINT/SIGTERM) from
    a normal non-zero exit.  Fix: separate _handle_signal() stores the
    signal name, cleanup re-raises with kill -SIG $$ after all cleanup
    completes, producing the standard 128+N exit code.  EXIT trap is now
    registered separately from signal traps to avoid masking the signal.
  * bump version 3.8.2 → 3.8.3

3.8.2  2026-03-16

  * fix CRITICAL: set -e/pipefail restoration order crash in run() and
    run_shell(): `$_prev_e && set -e` before `$_prev_pf && set -o pipefail`
    caused a latent crash — if caller had set -e ON but pipefail OFF, the
    `false && set -o pipefail` short-circuit returned 1 under the
    just-restored errexit, killing the script.  Fix: (a) use if/then
    instead of && to eliminate non-zero exit from false conditions,
    (b) restore pipefail BEFORE errexit for defense-in-depth.
