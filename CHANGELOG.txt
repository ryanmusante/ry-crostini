7.5.0  2026-03-29

  Bugs fixed:
  - replace p7zip-full (transitional in Trixie, 16.02+transitional.1)
    with 7zip (25.01+dfsg-1); p7zip-full will be removed in a future
    Debian release
  - replace qt5-style-plugins fallback (removed from Trixie) with
    adwaita-qt + adwaita-qt6; update step 11 verification to check
    adwaita-qt instead of qt5-style-plugins
  - fix Mesa shader cache silent misconfiguration: single-file Fossilize
    backend (MESA_DISK_CACHE_SINGLE_FILE=1) ignores MAX_SIZE — cache
    grows unbounded on 128 GB eMMC; switch to MESA_DISK_CACHE_DATABASE=1
    and reduce cap from 512M to 256M
  - replace deprecated vm.overrides={} in pipewire-pulse.conf.d with
    pulse.properties.rules match-action syntax (PipeWire 1.4.x); broaden
    KVM-only match (cpu.vm.name = "KVM") to any VM (cpu.vm.name = !null)
  - rename BOX64_DYNACACHE=1 to BOX64_DYNAREC_CACHE=1 in .box64rc;
    old name is aliased in Trixie's box64 0.3.4 but breaks on upstream
    0.3.8+
  - keep cros-guest-tools held permanently after dist-upgrade: Google
    has not published cros-im for Trixie; unholding breaks apt with
    unmet dependencies

  Improvements:
  - switch RetroArch audio from "pulse" to native "pipewire" driver
    (RetroArch 1.20+, Trixie); bump audio_latency from 64 to 96 to
    prevent underruns on SC7180P under gaming load
  - add stretch_mode=even-pixels as documented alternative in ScummVM
    config comment (ScummVM 2.9+)
  - add BOX64_DYNAREC_NATIVEFLAGS=1 to [default] and
    BOX64_DYNAREC_BIGBLOCK=3 to [wine] section in .box64rc
  - add mesa_glthread=true to gpu.conf (experimental — offloads GL call
    marshaling to separate CPU thread; Kryo 468 has headroom)
  - add systemd v257.* apt pin (/etc/apt/preferences.d/pin-systemd) to
    prevent accidental v258 upgrade; v258 refuses to run under cgroup v1
    which breaks Crostini containers
  - document Flatpak avoidance for gaming in README Known Limitations
    (triple sandbox overhead, Zink regression, RAM overhead, native .deb
    availability)
  - document box86 and box64 Box32 mode as 32-bit x86 alternatives in
    README x86 translation section
  - add WirePlumber 0.5 config format migration note to README Known
    Limitations (Lua → JSON .conf files)
  - update ChromeOS flag deprecation status in README:
    #crostini-multi-container deprecated at milestone 141 (was 140),
    #borealis-enabled deprecated (Borealis shut down 2026-01-01),
    #crostini-containerless (Baguette) new at ChromeOS 143+,
    #crostini-gpu-support and #exo-pointer-lock confirmed through
    ChromeOS 145

7.4.2  2026-03-28

  Fixes:
  - downgrade Sommelier preflight from warn() to log(): process is
    not running during install by design (started by container login,
    not by the running shell); message updated to "not yet active —
    will start on terminal restart ✓"
  - downgrade two cros.list NOTE messages from warn() to log():
    "cros.list resets on container restart" and "Debian repos in
    sources.list are permanent" are informational context, not errors
  - suppress spurious "Command exited 100" warn for unrar install:
    unrar (non-free) is unavailable on default Crostini sources and
    the failure is fully expected; replace run() call with a direct
    subshell that logs success or logs the fallback message without
    routing through run()'s generic non-zero-exit warn path
  - downgrade unrar step 11 check from ⚠/_verify_warn to
    ✓/_verify_pass when unar is present: unar handles RAR4/RAR5
    and GOG multi-part archives; absence of unrar is intentional
    and not a degraded state
  - fix dosbox-x emitting ⚠ "version unverified": remove dosbox-x
    from _TOOL_VER_FLAG empty-flag map; --version produces a parseable
    version string and the empty-flag workaround was never needed
  - fix glxinfo and gnome-disks emitting ⚠ "version unverified":
    both remain in _TOOL_VER_FLAG with flag=""; update check_tool()
    to treat an explicit empty flag entry as "installed, no version
    probe" and report ✓ (installed) / _verify_pass++ instead of
    ⚠ (version unverified) / _verify_warn++; preserves the ability
    to deliberately opt out of version probing for tools that have
    no usable version flag
  - net effect on a standard run: 0 [WARN] lines during execution
    (was 5); verification totals: ~75 passed / 1 warning (only the
    runtime Sommelier state check in step 11 summary, which is
    correct until terminal restart)
  - bump version to 7.4.2
